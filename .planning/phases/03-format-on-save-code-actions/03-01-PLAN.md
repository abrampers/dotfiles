Key lsp-mode APIs (from research, confirmed in lsp-mode.el):
- `lsp-format-buffer-on-save` (boolean, :local t) — enables lsp-format-buffer on save
- `lsp-format-buffer-on-save-list` (list of symbols) — if non-empty, only format buffers in this mode list
- `lsp-organize-imports` — interactive command sending source.organizeImports code action
- `lsp-completion-enable-additional-text-edit` (boolean, default t) — enables auto-import on completion

**Change 1: Add format-on-save settings to lsp-mode :custom section (around line 1087)**

In the `use-package lsp-mode` block, add two new `:custom` entries AFTER the existing `(lsp-diagnostics-provider :flycheck)` line and BEFORE the `:config` line:

```elisp
    (lsp-format-buffer-on-save t)
```

This enables format-on-save but ONLY for typescript-mode and typescriptreact-mode buffers. Go buffers are excluded because go-mode is not in the list. The lsp-mode code at line 5337-5340 checks `(member major-mode lsp-format-buffer-on-save-list)` before formatting.

**Change 2: Add organize-imports before-save hook to TypeScript use-package block (around line 1564)**

In the `use-package typescript-mode` block, add a `:config` addition that adds organize-imports on save for both TypeScript modes. Add these two lines AFTER the existing tree-sitter-major-mode-language-alist entries and BEFORE the `:init` keyword:

```elisp
    (add-hook 'typescript-mode-hook
              (lambda () (add-hook 'before-save-hook #'lsp-organize-imports nil t)))
    (add-hook 'typescriptreact-mode-hook
              (lambda () (add-hook 'before-save-hook #'lsp-organize-imports nil t)))
```

The `nil t` arguments mean: APPEND=nil (run before other hooks), LOCAL=t (buffer-local only — does NOT affect Go or other buffers).
