  <action>
Run the following terminal diagnostics to gather baseline information:

1. **Check lsp-mode git state** — Determine what commit the lockfile actually resolves to:
   ```bash
   # Check what the lockfile says
   grep "lsp-mode" emacs.d/straight/versions/default.el
   # Expected: ("lsp-mode" . "bdae0f406d040366193a359ab70f55af30e928ae")

   # Check if straight has cloned lsp-mode locally and what commit is checked out
   ls -la ~/.emacs.d/straight/repos/lsp-mode/.git 2>/dev/null && \
     git -C ~/.emacs.d/straight/repos/lsp-mode log --oneline -1
   # This shows the ACTUAL checked-out commit

   # Check what commit the v9.0.0 tag points to on GitHub
   git -C ~/.emacs.d/straight/repos/lsp-mode rev-parse v9.0.0 2>/dev/null || \
     echo "Tag v9.0.0 not found locally — will need to fetch"
   ```

2. **Check Node.js version via nvm**:
   ```bash
   # Source nvm and check version
   export NVM_DIR="$HOME/.nvm"
   [ -s "$NVM_DIR/nvm.sh" ] && source "$NVM_DIR/nvm.sh"
   node --version
   which node
   # Expected: v18+ or v20+ from nvm, path should contain .nvm/versions/

   # Check if typescript-language-server is already installed
   which typescript-language-server 2>/dev/null || echo "typescript-language-server NOT installed"
   ```

3. **Check current Emacs exec-path state** (non-interactive):
   ```bash
   # Run Emacs in batch mode to check exec-path
   emacs --batch --eval '(progn (load-file "~/.emacs.d/init.el") (message "exec-path: %s" exec-path) (message "node: %s" (executable-find "node")) (message "tls: %s" (executable-find "typescript-language-server")))' 2>&1 | grep -E "(exec-path|node|tls):"
   ```

Record all output for the checkpoint task.
  </action>
