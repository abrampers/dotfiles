
**Three changes explained:**

1. **Split auto-mode-alist** (D-05): Replace `"\\.tsx?\\'"` (matches both .ts and .tsx) with two entries:
   - `"\\.tsx\\'"` → `typescriptreact-mode` (TSX files)
   - `"\\.ts\\'"` → `typescript-mode` (TS files)
   The `.tsx` entry MUST come before the `.ts` entry in `add-to-list` calls so that `.tsx` matches first (since `add-to-list` prepends, the `.ts` call runs first but `.tsx` is added after, placing it earlier in the alist).

2. **Add typescript tree-sitter mapping** (D-06): Add `(add-to-list 'tree-sitter-major-mode-language-alist '(typescript-mode . typescript))` so plain .ts files use the `typescript` parser. The existing `typescriptreact-mode . tsx` line stays for .tsx files.

3. **Switch hooks to lsp-deferred** (D-07, D-08): Change both `#'lsp` to `#'lsp-deferred`, matching the go-mode pattern at config.org line 1250: `(go-mode . lsp-deferred)`. This prevents eager LSP startup before the buffer is ready.

**Also update the prose** above the code block (around line 1549-1551). The current prose says "we create a new derived mode that will map to both `.tsx` and `.ts`". Update to reflect the split:
